    • git推送时报错permission denied解决方法

使用git push git@github.com:[name]/[programname].github.git master时报错，提示没有权限 git@github.com: Permission denied (publickey). fatal: Could not read from remote repository.

Please make sure you have the correct access rights and the repository exists.

解决方法： 1.在项目根目录进入git bash 使用 $ ssh-keygen -t rsa -C "邮箱地址" 生成一个密钥 本地文件夹会生成一个密钥文件

2.进入github页面，在settings里点击ssh,点击新建ssh

3.完成以后本地使用ssh-add id_rsa（使用步骤1生成的的密钥文件,默认id_rsa）

ps:如果运行时报错：Could not open a connection to your authentication agent. 执行 eval $(ssh-agent)或者eval ssh-agent -s
